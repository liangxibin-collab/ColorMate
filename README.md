# 🎨 ColorMate - AI 照片追色工具

> 用数百张参考照片训练色彩风格模型，然后对普通照片进行智能追色。
> 全程在 Android 手机上运行，无需 PC。

## 技术方案

- **Android + Python（Chaquopy）**: Python核心算法嵌入Android应用
- **混合色彩迁移策略**（多重算法叠加，追求高质量）:
  1. **Lab统计匹配** - Reinhard色彩迁移算法的增强版
  2. **分位数映射** - 精确控制影调分布
  3. **色调直方图规定化** - H通道分布匹配
  4. **饱和度自适应映射** - 保护低饱和区域
  5. **自然度保护** - 边缘保护 + 色偏保护蒙版

## 功能

✅ 从目录中加载数百张JPG参考照片  
✅ 训练色彩风格模型（保存为.pkl文件）  
✅ 对单张目标照片追色  
✅ 批量追色  
✅ 导入/导出训练好的模型  
✅ 追色强度调节（0.0 ~ 1.0）  
✅ 高质量色彩迁移，自然度保护  

## 构建方式

### 环境要求

- Android Studio Hedgehog (2023.1.1) 或更新版本
- Android SDK 34
- ARM64设备（如荣耀MagicPad）

### 构建步骤

1. 在 Android Studio 中打开 `ColorMate/` 目录
2. 等待 Gradle 同步完成（首次需要下载 Python 依赖包）
3. 点击 Run 按钮

> ⚠️ **注意**: 首次构建时会下载 `opencv-python`、`numpy`、`scipy` 等包，需要网络连接。
> 构建后的 APK 大小约为 80-120MB。

### 或者直接使用命令行构建

```bash
cd ColorMate
./gradlew assembleDebug
```

APK 输出在 `app/build/outputs/apk/debug/`

## 使用方式

### 第1步：准备参考照片

- 收集 100~500 张你喜欢的色彩风格的照片（JPG格式）
- 把它们放在手机上的一个目录里
- 照片主题越一致，追色效果越好（比如全是胶片风、全是日系清新等）

### 第2步：训练模型

1. 打开 ColorMate
2. 点击「选择训练目录」→ 选择放参考照片的目录
3. 确认数量后点击「开始训练」
4. 等待训练完成

### 第3步：追色

1. 点击「选择目标照片」
2. 调整强度滑块（默认1.0最强效果）
3. 点击「开始追色」
4. 追色结果自动保存到 `Pictures/ColorMate/` 目录

### 模型管理

- **导出模型**: 训练完成后模型默认保存在 app 缓存目录，可通过「导入模型」分享
- **导入模型**: 点击「导入已有模型」可以加载之前训练好的模型

## 算法说明

ColorMate 采用混合色彩迁移策略，依次执行多层调色，力求最佳效果：

### 第1层：Lab统计匹配
将目标照片的 Lab 色彩空间均值和标准差对齐到参考风格。这是最基础的全局色调调整。

### 第2层：分位数映射
精确匹配每个通道的亮度/色彩分布曲线。色调整控制更精细。

### 第3层：色调直方图匹配
调整HSV空间的H通道，使色调分布（如"偏暖""偏冷"）接近参考风格。

### 第4层：饱和度映射
调整饱和度均值和分布，适配参考风格的饱和度特征。

### 第5层：自然度保护
- **色偏保护**：防止色偏过大的区域失真
- **边缘保护**：保留原图的细节和纹理
- **混合蒙版**：在低强度时更保守，确保不会产生不自然的颜色

## 常见问题

**Q: 为什么追色结果看起来偏色严重？**
A: 尝试调低强度滑块（0.5~0.7）。如果参考照片风格非常极端（如高饱和）
且目标照片差异很大，建议使用稍低的强度。

**Q: 训练需要多少张照片？**
A: 建议至少 50 张，300-500 张效果最佳。照片越多，风格提炼越准确。

**Q: 支持 PNG 格式吗？**
A: 目前仅支持 JPG/JPEG 格式。PNG 需要额外处理。

**Q: 处理速度如何？**
A: 训练速度取决于照片数量和分辨率，500张约1-3分钟。
单张追色通常在 1-5 秒内完成。
